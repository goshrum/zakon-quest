# Content Policy

Know the Law teaches law, so **accuracy is the project's highest priority**. This
policy governs every question in the dataset.

## Principles

1. **Cite everything.** Every question carries a `citation` to a specific,
   verifiable norm. No citation, no merge.
2. **Prefer stable facts.** Favor well-established, slow-to-change norms (which
   code governs a matter; iconic article numbers; foundational principles) over
   volatile specifics (exact monetary thresholds, rates) that change between
   editions. When a precise number is unstable, ask about the governing code or
   branch instead.
3. **Educational, not advice.** The app is a learning tool and explicitly **not
   legal advice**. This is stated in the UI and the README.
4. **Verifiability over volume.** A smaller bank of rock-solid, sourced questions
   beats a large bank of shaky ones.

## Sources

Authoritative sources include the official legal portal
(`pravo.gov.ru`) and recognized legal references. The `citation` field should
name the norm (article + code); issues/PRs adding content should link a source
URL.

## Reporting a wrong question

Open an issue with the **Question correction** template, include the question
`id`, the problem, the correct citation, and a source URL. Citation-backed
corrections take priority and are typically fast-tracked.

## Translation & jurisdiction note

The current dataset describes the **Russian** legal system, expressed entirely in
English. Translations change wording only — never facts, article numbers, or
which code applies. Other jurisdictions are welcome as **separate** datasets; see
[`adding-a-jurisdiction.md`](adding-a-jurisdiction.md).
